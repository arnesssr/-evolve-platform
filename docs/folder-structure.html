ğŸ› ï¸ Working with the Structure

Adding New Features

When adding new functionality, follow this pattern:

  1. Create models in the appropriate models/ directory
  2. Add business logic in services/
  3. Create data access methods in repositories/
  4. Build API endpoints in api/v1/
  5. Create web views in views/ if needed
  6. Add templates in the appropriate templates/ subdirectory
  7. Write tests in tests/

Quick Navigation Tips

  • Looking for business logic? Check App/[module]/services/
  • Need to modify database queries? Look in repositories/
  • Working on APIs? Navigate to api/v1/
  • Updating UI? Templates are in global templates/
  • Configuring the app? Check config/settings.py
  • Adding dependencies? Update requirements.txt
âœ… Best Practice: Always follow the established folder structure when adding new features. This keeps the codebase organized and makes it easier for team members to find and understand code.
