üöÄ Getting Started

This guide will help you set up your local development environment and get the Evolve Payments Platform running on your machine.

Required Software

Python 3.8 or higher Backend runtime python.org

‚öôÔ∏è Environment Setup

1Ô∏è‚É£ Clone the Repository

# Clone the repository
git clone https://github.com/JoyNyayieka/evolve-payments-platform.git

# Navigate to project directory
cd evolve-payments-platform

2Ô∏è‚É£ Create Virtual Environment

Always use a virtual environment to isolate project dependencies.

Windows (PowerShell):

# Create virtual environment
python -m venv evolve_env

# Activate virtual environment
.\evolve_env\Scripts\Activate.ps1

# Verify activation (should show virtual env name in prompt)
# (evolve_env) PS C:\path\to\project>

macOS/Linux:

# Create virtual environment
python3 -m venv evolve_env

# Activate virtual environment
source evolve_env/bin/activate

# Verify activation
# (evolve_env) user@machine:~/project$
‚ö†Ô∏è Important: Always activate the virtual environment before working on the project. You'll need to do this every time you open a new terminal session.

3Ô∏è‚É£ Install Dependencies

# Make sure virtual environment is activated
# Install all required packages
pip install -r requirements.txt

# Verify installation
pip list

Current dependencies include:

  • Django 5.2.4 (web framework)
  • python-decouple 3.8 (environment configuration)
  • requests 2.32.4 (HTTP client for API integrations)

4Ô∏è‚É£ Configure Environment Variables

Create a .env file in the root directory with the following configuration:

# SMS Configuration (SMSLeopard)
SMSLEOPARD_ENVIRONMENT=sandbox
SMSLEOPARD_API_KEY=your_api_key_here
SMSLEOPARD_API_SECRET=your_api_secret_here
SMSLEOPARD_ACCESS_TOKEN=your_access_token_here

# Email Configuration
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_HOST_USER=your_email@gmail.com
EMAIL_HOST_PASSWORD=your_app_password

# Pesapal Configuration (Payment Gateway)
PESAPAL_CONSUMER_KEY=your_consumer_key
PESAPAL_CONSUMER_SECRET=your_consumer_secret
üîê Security Note: Never commit the .env file to version control. It's already in .gitignore. Get the actual values from your team lead or project documentation.

5Ô∏è‚É£ Setup Database

The project uses SQLite for development (no additional setup required).

# Apply database migrations
python manage.py migrate

# Create superuser account
python manage.py createsuperuser

# Follow prompts to create admin account

6Ô∏è‚É£ Run Development Server

Option 1: Using PowerShell Script (Windows)

# Run the server using the provided script
.\run_server.ps1

Option 2: Manual Command

# Run development server manually
python manage.py runserver

# Server will start at http://127.0.0.1:8000/
‚úÖ Success! If everything is working correctly, you should see the Django development server running and be able to access the application at http://127.0.0.1:8000/

üîÑ Development Workflow

Daily Development Commands

# Start your development session

üõ†Ô∏è Common Development Tasks

Adding New Dependencies

  1. Install the package: pip install package_name
  2. Test your application to make sure it works
  3. Update requirements.txt: pip freeze > requirements.txt
  4. Or manually add: package_name==version
  5. Commit both your code and the updated requirements.txt

Working with Models

# After creating or modifying models
python manage.py makemigrations
python manage.py migrate

# To see migration SQL (optional)
python manage.py sqlmigrate app_name migration_number

Testing

# Run all tests
python manage.py test

# Run tests for specific app
python manage.py test App.admin

# Run tests with verbosity
python manage.py test --verbosity=2

# Run specific test class
python manage.py test App.admin.tests.TestUserModel

üîß Troubleshooting

Common Issues

‚ùå "No module named 'decouple'"

Solution: Make sure virtual environment is activated and dependencies are installed:

.\evolve_env\Scripts\Activate.ps1  # Windows
pip install -r requirements.txt

‚ùå "django.core.exceptions.ImproperlyConfigured"

Solution: Check your .env file exists and has the required variables.

‚ùå "Permission denied" on PowerShell

Solution: Enable PowerShell execution policy:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

‚ùå Database errors

Solution: Reset database (development only):

# Delete db.sqlite3 file
rm db.sqlite3

# Run migrations again
python manage.py migrate
python manage.py createsuperuser
